# Heurísticas para Identificação de Requisitos em Entrevistas

Durante entrevistas, stakeholders raramente descrevem requisitos de forma formal.
Normalmente eles descrevem problemas, necessidades ou atividades do dia a dia.

O analista de requisitos deve interpretar essas falas e identificar possíveis requisitos.

## Indicadores de possíveis requisitos funcionais

Frases que indicam funcionalidades desejadas:

- “preciso registrar…”
- “o sistema deveria permitir…”
- “seria bom se pudesse…”
- “precisamos consultar…”
- “tem que gerar relatório…”
- “o usuário deve conseguir…”

Essas expressões podem indicar um **requisito funcional**.

---

## Indicadores de requisitos não funcionais

Frases relacionadas a qualidade do sistema:

- “tem que ser rápido”
- “não pode travar”
- “tem que ser seguro”
- “precisa funcionar no celular”
- “tem que ser fácil de usar”

Essas expressões indicam **requisitos não funcionais**.

---

## Indicadores de requisitos implícitos

Algumas frases descrevem problemas atuais e indicam necessidades implícitas.

Exemplos:

“Hoje fazemos tudo manualmente.”

Possível requisito implícito:
O sistema deve automatizar esse processo.

---

“Demoramos muito para encontrar os dados.”

Possível requisito implícito:
O sistema deve permitir busca rápida de informações.

---

## Regra importante

Sempre justificar o requisito identificado com evidência textual da entrevista.
Nunca inventar requisitos que não possam ser relacionados ao texto analisado.
