# Prompts modelo (copiar e colar)

Use como templates para análise no chat.

---

## A) Compreensão do domínio

Aja como um analista de requisitos experiente.

Estou iniciando um projeto de desenvolvimento de software para o seguinte domínio:

[descrever domínio]

Com base nesse domínio, gere uma lista de perguntas que devem ser feitas aos stakeholders para compreender:

- contexto organizacional
- processos atuais
- problemas existentes
- necessidades do negócio

Organize as perguntas por categorias.

---

## B) Identificação de stakeholders

Aja como um gerente de projetos com experiência em Engenharia de Requisitos.

Contexto:
Estou iniciando um projeto para desenvolver um sistema para o seguinte setor:

[descrever setor]

Tarefa:

Identifique os stakeholders relevantes para a fase de elicitação de requisitos.

Para cada stakeholder explique:

- por que ele é importante
- quais informações ele pode fornecer
- quais técnicas de elicitação são mais adequadas

---

## C) Extração de Requisitos Funcionais

Tarefa:

Analise a transcrição e identifique os requisitos funcionais mencionados.

Para cada requisito informe:

- identificador (RF-01, RF-02, etc.)
- descrição clara
- se é explícito ou implícito
- trecho da entrevista que comprova o requisito
